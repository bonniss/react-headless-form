---
title: array
---

`array` represents a repeatable list of fields where each item shares the same shape. It is backed by RHF's `useFieldArray` under the hood.

```tsx
addresses: {
  type: "array",
  label: "Addresses",
  props: {
    config: defineConfig<Address>({
      street: { type: "text", label: "Street" },
      city:   { type: "text", label: "City" },
    }),
  },
}
```

`props.config` defines the shape of each item. BlueForm handles namespace scoping automatically — a `street` field in item 2 is registered as `addresses.2.street` without any manual wiring.

## Rendering

Array fields require a renderer — either `render` inline or a component registered in `fieldMapping["array"]`.

### Via `render`

The `render` function is responsible for the shell — the container, action buttons, and error message. It receives the full [`useArrayField`](../../field-authoring/use-array-field.mdx) context as arguments, so no hook call is needed:

```tsx
addresses: {
  type: "array",
  render: ({ label, errorMessage, items, append, renderItems }) => (
    <fieldset>
      <legend>{label}</legend>
      {renderItems()}
      <button type="button" onClick={() => append({ street: "", city: "" })}>
        Add
      </button>
      {errorMessage && <div>{errorMessage}</div>}
    </fieldset>
  ),
  props: {
    config: defineConfig<Address>({ ... }),
  },
}
```

For per-item control — individual remove buttons, item wrappers — use `renderItem` and `items` directly:

```tsx
render: ({ label, items, append, renderItem, remove }) => (
  <fieldset>
    <legend>{label}</legend>
    {items.map((field, index) => (
      <div key={field.id}>
        {renderItem(field, index)}
        <button type="button" onClick={() => remove(index)}>Remove</button>
      </div>
    ))}
    <button type="button" onClick={() => append({})}>Add</button>
  </fieldset>
),
```

### Via `fieldMapping`

For a consistent array shell across all forms, register a global component via `fieldMapping["array"]`. It uses `useArrayField` directly:

```tsx
const ArrayField = () => {
  const { label, errorMessage, items, renderItems, append, pop } = useArrayField()

  return (
    <fieldset>
      <legend>{label}</legend>
      {renderItems()}
      <button type="button" onClick={() => append({})}>Add</button>
      <button type="button" disabled={!items.length} onClick={pop}>Remove last</button>
      {errorMessage && <div>{errorMessage}</div>}
    </fieldset>
  )
}

const [Form] = setupForm({
  fieldMapping: defineMapping({
    text: InputField,
    array: ArrayField,
  }),
})
```

Fields with no `render` automatically fall back to `fieldMapping["array"]`. If neither is provided, an error is thrown at render time.

## Array-level validation

`rules` on an array field apply to the array itself — not individual items. `errorMessage` carries the array-level error:

```tsx
addresses: {
  type: "array",
  rules: {
    minLength: { value: 2, message: "At least 2 addresses required" },
  },
  render: ({ errorMessage, renderItems, append }) => (
    <div>
      {renderItems()}
      <button type="button" onClick={() => append({})}>Add</button>
      {errorMessage && <div>{errorMessage}</div>}
    </div>
  ),
}
```

Item-level validation is defined in `props.config` and handled the same way as regular fields. Use `errorAt(index)` when you need to inspect an item's error state from outside the item — for example, to highlight a collapsed item that contains invalid fields.

## Complete example

```tsx
addresses: {
  type: "array",
  label: "Addresses",
  rules: {
    minLength: { value: 2, message: "At least 2 addresses required" },
  },
  props: {
    config: defineConfig<Address>({
      street:  { type: "text", label: "Street" },
      city:    { type: "text", label: "City" },
      country: { type: "text", label: "Country" },
    }),
  },
  render: ({ label, errorMessage, items, renderItems, append, duplicate, pop, clear }) => (
    <fieldset>
      <legend>{label}</legend>

      {renderItems()}

      <div>
        <button
          type="button"
          onClick={() => append({ street: "", city: "", country: "" })}
        >
          Add
        </button>
        <button
          type="button"
          disabled={!items.length}
          onClick={() => duplicate(items.length - 1)}
        >
          Duplicate last
        </button>
        <button type="button" disabled={!items.length} onClick={pop}>
          Remove last
        </button>
        <button type="button" disabled={!items.length} onClick={clear}>
          Clear all
        </button>
      </div>

      {errorMessage && <div>{errorMessage}</div>}
    </fieldset>
  ),
}
```
