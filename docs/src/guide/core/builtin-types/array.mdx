# Array

`array` represents a repeatable group of fields — a list of items that share the same shape. It is backed by RHF's `useFieldArray` under the hood.

```tsx
addresses: {
  type: "array",
  label: "Addresses",
  props: {
    config: defineConfig<Address>({
      street: { type: "text", label: "Street" },
      city: { type: "text", label: "City" },
    }),
  },
}
```

The `props.config` defines the shape of each item. BlueForm handles scoping automatically — fields inside each item are registered under the correct namespace (e.g. `addresses.0.street`, `addresses.1.city`).

## Rendering array items

The array's `render` function is responsible for the **shell** — the container, the add/remove buttons, the error message. It does not render individual items directly.

Individual items are rendered using `renderItems()`, a helper from `useArrayField` that iterates over the current list and renders each item using `props.config` as the field definition. Each item is automatically scoped to its index — so a `street` field in item 2 is registered as `addresses.2.street` without any manual wiring.


```tsx
render: () => {
  const { items, renderItems, append } = useArrayField()

  return (
    <div>
      {/* renders each item using props.config */}
      {renderItems()}

      <button type="button" onClick={() => append({ street: "", city: "" })}>
        Add address
      </button>
    </div>
  )
}
```

`renderItems()` is shorthand for `items.map(renderItem)`. Use `renderItem(field, index)` directly when you need per-item control — for example, to wrap each item in a card with its own remove button:

```tsx
{items.map((field, index) => (
  <div key={idAt(index)}>
    {renderItem(field, index)}
    <button type="button" onClick={() => remove(index)}>Remove</button>
  </div>
))}
```

## Array methods

All of RHF's [`useFieldArray`](https://react-hook-form.com/docs/usefieldarray) methods are available:

| Method | Description |
|---|---|
| `append(value)` | Add item at the end |
| `prepend(value)` | Add item at the beginning |
| `insert(index, value)` | Add item at a specific position |
| `remove(index?)` | Remove item by index, or all items if omitted |
| `move(from, to)` | Move item to a new position |
| `swap(a, b)` | Swap two items |
| `update(index, value)` | Replace item at index |
| `replace(values)` | Replace the entire list |

BlueForm also provides convenience helpers:

| Helper | Description |
|---|---|
| `push(value)` | Alias of `append` |
| `pop()` | Remove the last item |
| `clear()` | Remove all items |
| `duplicate(index)` | Append a copy of the item at `index`, using current form values |
| `idAt(index)` | Return the stable RHF key for the item (fallback to index) |
| `errorAt(index)` | Return the validation error subtree for the item at `index` |

## Array-level validation

Validation rules on the array field apply to the array itself, not individual items. Use `fieldProps.errorMessage` to display them:

```tsx
addresses: {
  type: "array",
  rules: {
    minLength: { value: 2, message: "At least 2 addresses required" },
  },
  render: () => {
    const { fieldProps, renderItems } = useArrayField()
    return (
      <div>
        {renderItems()}
        {fieldProps.errorMessage && (
          <div>{fieldProps.errorMessage}</div>
        )}
      </div>
    )
  },
}
```

Each item's fields define their own validation rules via `props.config`, and errors are handled the same way as regular fields. `errorAt(index)` exists for the cases where you need to inspect an item's error state from *outside* the item itself, such as highlighting a collapsed or minimized item that contains invalid fields.

## Complete example

```tsx
addresses: {
  type: "array",
  label: "Addresses",
  rules: {
    minLength: { value: 2, message: "At least 2 addresses required" },
  },
  props: {
    config: defineConfig<Address>({
      street: { type: "text", label: "Street" },
      city: { type: "text", label: "City" },
      country: { type: "text", label: "Country" },
    }),
  },
  render: ({ fieldProps }) => {
    const { items, renderItems, append, duplicate, pop, clear } = useArrayField()

    return (
      <fieldset>
        <legend>{fieldProps.label}</legend>

        {renderItems()}

        <div>
          <button
            type="button"
            onClick={() => append({ street: "", city: "", country: "" })}
          >
            Add
          </button>
          <button
            type="button"
            disabled={!items.length}
            onClick={() => duplicate(items.length - 1)}
          >
            Duplicate last
          </button>
          <button
            type="button"
            disabled={!items.length}
            onClick={pop}
          >
            Remove last
          </button>
          <button
            type="button"
            disabled={!items.length}
            onClick={clear}
          >
            Clear all
          </button>
        </div>

        {fieldProps.errorMessage && (
          <div>{fieldProps.errorMessage}</div>
        )}
      </fieldset>
    )
  },
}
```

## Using fieldMapping for a global array component

Instead of defining `render` inline on every array field, you can register a global array component via `fieldMapping`. This is useful when all array fields in your app share the same shell UI — add button, remove button, error display — and only differ in their `props.config`.

```tsx
const ArrayField = () => {
  const { fieldProps, items, renderItems, append, pop } = useArrayField()

  return (
    <fieldset>
      <legend>{fieldProps.label}</legend>
      {renderItems()}
      <button type="button" onClick={() => append({})}>Add</button>
      <button type="button" disabled={!items.length} onClick={pop}>Remove last</button>
      {fieldProps.errorMessage && <div>{fieldProps.errorMessage}</div>}
    </fieldset>
  )
}

const [Form] = setupForm({
  fieldMapping: defineMapping({
    text: InputField,
    array: ArrayField,   // registered once, used everywhere
  }),
})
```

Array fields with no `render` automatically fall back to `fieldMapping["array"]`:

```tsx
config={{
  addresses: {
    type: "array",
    label: "Addresses",
    props: {
      config: defineConfig<Address>({
        street: { type: "text", label: "Street" },
        city: { type: "text", label: "City" },
      }),
      // no render — uses fieldMapping["array"]
    },
  },
  tags: {
    type: "array",
    label: "Tags",
    props: {
      config: defineConfig<Tag>({
        name: { type: "text", label: "Name" },
      }),
    },
  },
}}
```

:::tip
If neither `render` nor `fieldMapping["array"]` is provided, a warning will be issued that the array field has no renderer.
:::
