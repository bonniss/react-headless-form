## Examples

| # | Example | What it covers |
|---|---|---|
| 1 | [Login Form](./foundational/login-form.mdx) | Flat model, basic validation, `props` passthrough |
| 2 | [Registration Form](./foundational/registration-form.mdx) | Multiple field types, cross-field validation, `disabled` as function |
| 3 | [Nested Form](./foundational/nested-form.mdx) | `section` + `nested: true`, `defineConfig`, reusable config fragments |
| 4 | [Array Fields](./foundational/array-form.mdx) | `array` type, `useArrayField`, add / remove / duplicate |
| 5 | [Conditional Fields](./foundational/conditional-form.mdx) | `visible` as function, conditional validation with `validate` |
| 6 | [Multi-step Wizard](./foundational/wizard-form.mdx) | `section` composition, shared RHF state across steps, `Section` component |

:::note
The forms in these examples are styled via shared CSS applied to the field components and wrapper. BlueForm itself is fully headless — it ships no styles, no UI primitives, and has no opinion on how your fields look.
:::
